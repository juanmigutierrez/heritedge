Design a modern mobile app UI/UX concept for a heritage exploration experience in Piazza Duomo, Milano. The app should feel elegant, cultural, immersive, and easy to use with one hand outdoors. The visual style should be clean, premium, and minimal, with strong readability, soft cards, large touch targets, clear icons, and a museum-quality aesthetic. Prioritize usability in short sessions, noisy outdoor environments, and quick interactions.

App concept:
The app helps visitors explore the heritage of Piazza Duomo through 3 experiences:
1. Quick introductory heritage information about the Duomo
2. AR exploration of Piazza Duomo across historical time periods
3. A mobile treasure hunt / quiz comparing past and present artifacts

Main places:
- Duomo di Milano
- Galleria
- Palazzo

Main modalities:
- Touch
- Chat with AI
- Optional voice guide / voice commands
- AR visualization

Design a full mobile app flow with these screens:

1. Welcome / Home screen
- Title: Explore Piazza Duomo Through Time
- Short subtitle explaining heritage discovery
- Main CTA buttons:
  - Start Quick Guide
  - Enter AR Experience
  - Start Treasure Hunt
- Secondary access to chat assistant
- Elegant hero image or illustration inspired by Piazza Duomo

2. Task 1: Heritage Quick Guide screen
- Show concise introductory information about Duomo heritage
- Information should be scannable and readable in under 1 minute
- Include cards like:
  - Past
  - Present
  - Why it matters
- Include:
  - Read more
  - Ask AI
  - Play voice guide
- Add suggested question chips for user prompts
- Keep layout optimized for one-handed use

3. Task 1: AI Chat screen
- Conversational interface for asking about the heritage site
- Suggested prompts such as:
  - Tell me about the Duomo history
  - What changed over time?
  - Why is this place important?
- If user asks something out of scope, show a graceful limitation message and suggest related topics
- Include voice input button and text input
- Include trust-building UX: “I can answer about the selected heritage site and related curated topics”

4. Task 2: AR overview screen
- Show camera-based AR mode with a miniature reconstruction of Piazza Duomo placed on a table/floor
- Landmarks visible in small scale:
  - Duomo
  - Galleria
  - Palazzo
- Include top status indicators:
  - Active landmark
  - Selected time period
- Include bottom controls:
  - Change period
  - Voice command
  - Help
  - Exit AR
- Make the AR UI simple, immersive, and uncluttered

5. Task 2: Artifact detail screen in AR
- After selecting a landmark, zoom into the artifact
- Show:
  - Artifact name
  - Time period
  - Curated facts
  - Visual comparison
  - Optional voice playback
- Include tabs or chips for different topics:
  - Exterior
  - Construction history
  - Cultural meaning
- Add a “Back to Piazza overview” button

6. Time period selection UI
- A very clear and visually rich selector for 3 historical states:
  - Medieval
  - After War
  - Present
- Switching time periods should update both visuals and facts
- Show past-present comparison in a simple, intuitive way

7. Voice command interaction screen/state
- Show push-to-talk interaction
- Example command:
  - “Take me to Duomo in medieval times”
- Show live transcription
- If confidence is high, execute directly
- If ambiguous, ask for confirmation with a friendly grounding prompt
- Example:
  - “Did you mean Duomo in the 15th century?”
- Design a clear confirmation interface

8. Error / grounding recovery states
Design user-friendly UI for these situations:
- Two landmarks are ambiguous in AR view
- Voice input is unclear because of noise
- Content requested is out of scope
- AR tracking fails due to poor lighting or unstable surface
For each case, provide:
- Clear explanation
- Suggested recovery action
- Touch fallback option
- Never leave the user at a dead end

9. Task 3: Treasure Hunt / Quiz screen
- Outdoor mobile quiz experience about heritage artifacts
- Users compare present and past versions of Duomo, Galleria, and Palazzo
- Support voice answers and touch-based multiple-choice fallback
- Include controls like:
  - Hint
  - Repeat
  - Skip
- Show progress indicator
- Keep the experience playful but educational

10. Task 3: Quiz feedback screen
- After each answer, show:
  - Correct / incorrect feedback
  - Short historical explanation
  - Optional deeper insight
- Make feedback motivating and concise

11. Final summary / completion screen
- Summarize what the user explored
- Show visited landmarks
- Show time periods explored
- Show quiz progress/results
- CTA options:
  - Explore again
  - Continue to another place
  - Ask AI more questions

UX requirements:
- Mobile-first design
- Outdoor readability
- High contrast and large text
- One-handed interaction
- Session length around 5–15 minutes
- Clear task progression
- Minimal cognitive load
- Elegant and culturally respectful visuals
- Strong sense of trust and clarity
- Easy switching between browse, chat, voice, and AR
- Include fallback interactions for voice and AR failures

Grounding and interaction design principles:
- Clarify uncertainty instead of guessing
- Use visual highlighting when references are ambiguous
- Always show current system state: active landmark + selected time period
- Suggest alternatives when content is unavailable
- Reduce cognitive load with concise information and progressive disclosure
- Maintain consistency across all tasks

5. A short explanation of how the UX supports trust, ambiguity reduction, and one-handed outdoor use
